## Aperçu

Au cours de ce tutoriel, vous allez construire un jeu d'aventure donjon alimenté par l'IA. Ce tutoriel ne nécessite aucune connaissance préalable du `@aws/nx-plugin` ou des technologies associées. Les techniques que vous apprendrez ici sont fondamentales pour créer n'importe quelle application basée sur le `@aws/nx-plugin`, et leur maîtrise vous donnera une compréhension approfondie de cet outil ainsi qu'une connaissance de base de [NX](https://nx.dev/).

<Aside>Ce tutoriel est conçu pour ceux qui préfèrent apprendre en pratiquant et souhaitent rapidement créer quelque chose de concret. Si vous préférez apprendre chaque concept étape par étape, consultez les guides individuels des composants.</Aside>

À l'issue de ce tutoriel, vous serez capable de :

- utiliser le `@aws/nx-plugin` pour créer de nouvelles applications
- utiliser NX pour gérer/construire votre base de code
- construire des APIs avec tRPC et FastAPI
- utiliser Tanstack router pour créer de nouvelles pages
- utiliser Tanstack query pour appeler des APIs backend
- modifier du code généré pour l'adapter à vos besoins
- créer et déployer une infrastructure CDK

### Composants de l'application

L'architecture du jeu utilise les composants suivants :

<Image class="white-bg" src={dungeonAdventureArchitecturePng} alt="Architecture du jeu donjon" width="800" height="600" />

- Une API [tRPC](https://trpc.io/) utilisant [ElectroDB](https://electrodb.dev/)/[DynamoDB](https://aws.amazon.com/dynamodb/) pour gérer l'état du jeu
- Une fonction Lambda Python/[FastAPI](https://fastapi.tiangolo.com/) avec streaming de réponses pour la génération d'histoire (alimentée par [Amazon Bedrock](https://aws.amazon.com/bedrock/))
- Un site React/Vite frontend utilisant :
  - Cognito/Identity Pools pour les appels API sécurisés
  - [Tanstack router](https://tanstack.com/router/latest) avec routage basé sur les fichiers et typage fort
  - Des SDK générés pour appeler les APIs de jeu et d'histoire (utilisant [Tanstack query](https://tanstack.com/query/latest))
- Une infrastructure [CDK](https://docs.aws.amazon.com/cdk/v2/guide/home.html) pour le déploiement
